## Contexto

Uma empresa de educação quer criar pequenos jogos em Java para ensinar lógica de programação aos iniciantes. Esses jogos também servirão como exemplo de boas práticas de documentação para outros estudantes.

Você foi convidado para criar um jogo simples de adivinhação de número, totalmente em console, e precisa documentar o código usando Javadoc, para que outros desenvolvedores entendam facilmente o funcionamento do jogo.

 

## Objetivos

Implementar um jogo simples em Java usando classes
Separar as responsabilidades em classes
Documentar o código utilizando Javadoc:
Classes
Atributos
Métodos
Parâmetros e retornos
Exceções (se houver)
Gerar documentação HTML
Analisar a importância da documentação mesmo em projeto simples
 

## Descrição do jogo

O jogo "Adivinha o Número" funciona assim:

O computador escolhe um número aleatório entre 0 e 100
O jogador tenta adivinhar o número
A cada tentativa, o jogo informa se o palpite é maior, menor ou igual ao número secreto. Pode usar também "quente", "morno" ou "frio"
O jogo conta quantas tentativas o jogador fez
Quando o jogador acerta, o jogo exibe uma mensagem de parabéns e o número de tentativas
